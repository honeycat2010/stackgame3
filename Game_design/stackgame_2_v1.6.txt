빵 스택 게임 기획서 v1.6 (C++ / Steam 배포 전제)


0. 목표
- C++로 제작, Windows Steam 배포 가능 구조
- 화면비 2:3 세로형 유지 (레터박스 허용)
- 상태 흐름 고정: Title → Menu → InGame → GameOver
- 설정 메뉴 항목: BGM On/Off, FX On/Off, 화면(Windowed, Fullscreen)
- 디버그/밸런싱을 위해 스테이지 파일을 별도 파일로 관리


1. 해상도/화면 정책

1.1 내부 기준 해상도 (논리 좌표)
- 기본 출력: 480×720 (2:3)
- 모든 UI/게임 좌표는 이 내부 해상도를 기준으로 작성
- 빵이 스택되어 일정 높이 이상 오면 빵의 가장 윗부분과 HUD만 남기고 화면을 스크롤
- 스크롤 대상: 배경, 스택된 빵

1.2 출력 방식
- 윈도우 크기 = 480*scale × 720*scale
- 화면비가 다른 디스플레이에서 전체화면 시:
  - 레터박스(좌우 또는 상하 여백) 허용
  - 게임 영역은 항상 2:3 유지

1.3 디스플레이 모드
- displayMode: Windowed / Fullscreen (설정에서 변경 가능)


2. 상태(씬) 정의

2.1 Title
- 표시: 게임 타이틀 로고/텍스트
- 입력:
  - Enter/Space: Menu로 이동
  - Esc: 종료

2.2 Menu
- 항목: 게임 시작 / 설정 / 종료
- 이동: ↑↓  선택: Enter  뒤로: Esc(Title로)

2.3 Settings (Menu 내부 서브 화면)
- 항목:
  1) BGM: On/Off
  2) FX: On/Off
  3) 화면: Windowed / Fullscreen
- 조작:
  - ↑↓: 항목 이동
  - ←→: 값 변경 (토글/옵션 순환)
  - Esc: 저장 후 Menu로 복귀
- 저장 정책:
  - 값 변경 즉시 반영 (사운드/화면)
  - Settings 종료 시 config.json 저장

2.4 InGame
- HUD:
  - Stage
  - PlacedCount/Target (targetPlaced)
  - total (누적 스택 개수)
  - miss (현재 miss 수/3) : miss 3회 시 게임 종료
- 입력:
  - Left/Right: 캐쳐 X축 자유 이동
  - Esc: 일시정지 메뉴 진입
- 게임 루프:
  - 빵 스폰 → 낙하 → 캐치/미스 판정 → 탑 적재 → 일정 탑 적재 시 화면 스크롤 → 스테이지 진행/게임오버

- 일시정지(Pause) 규칙:
  - Esc 입력 시 InGame 내부 모달 Pause 상태로 전환
  - Pause 상태에서는 빵 스폰/낙하/스택/스크롤 업데이트 전부 정지
  - 입력은 Pause 메뉴에만 전달
  - Resume 선택 시 3초 카운트다운 후 InGame 복귀
  - Pause는 별도 Scene으로 분리하지 않음 (상태 구조 단순 유지)

2.5 GameOver
- 표시: Game Over 이미지(png) 3초 출력 후 메뉴 이동, 최종 점수/스테이지 출력
- 메뉴:
  - 재시작 (InGame 새로 시작)
  - 타이틀로 (Title)
  - 종료


3. 게임 규칙

3.1 레인 시스템 (스폰 전용)
- 레인은 총 5개이며 캐쳐 이동 기준이 아님
- 레인은 오직 빵 스폰 X 좌표 프리셋 배열
- 정의: 화면 폭 기준 균등 분포 (레인 중심)
- 스폰 시 랜덤 레인을 선택하여 해당 X에서 낙하

3.2 캐쳐 이동 (자유 이동)
- 캐쳐는 연속 좌표 기반 이동
- 입력: Left / Right 키만 사용

3.3 캐치 판정
- 판정 방식: 단일 점 판정
- 판정점: 빵 이미지의 중심점 (centerX = imageWidth/2, centerY = imageHeight/2)
- 판정 절차:
  1. 빵 bottomY가 캐쳐 기준선(screen-space 고정 Y)에 도달한 프레임에 1회 수행
  2. 빵 중심점을 월드 좌표로 변환 후 캐쳐 히트맵 좌표계로 변환
  3. 해당 좌표의 히트맵 색상이 (R255, G0, B255)이면 캐치 성공
- 매 프레임 중복 판정 금지 (좌표계 혼용 방지)

3.4 적재 위치 기록
- 캐치 성공 시 탑에 블록 추가, 누적 카운트(total) 증가
- total이 15의 배수가 될 때마다 화면 스크롤 발생 (→ 4.3 스크롤 동작 참조)


4. 좌표계 및 스크롤 정책

4.1 좌표계 구분
- World 좌표 적용 객체:
  - 스택된 빵
  - 낙하 중 빵
  - 배경
  - 렌더 시에만 screenY = worldY - camY 적용

- Screen 좌표 고정 객체:
  - 캐쳐
  - HUD
  - Pause UI
  - GameOver UI
  - camY의 영향을 받지 않음

4.2 camY 적용 규칙
- camY는 월드 객체에만 적용
- 캐쳐 및 모든 UI 요소에는 camY 적용 안 함
- ScreenToWorld 보정 계산을 캐쳐에 사용하지 않음
- camY 값 변경은 스크롤 트리거(total 15의 배수)에서만 발생

4.3 스크롤 동작
- 스크롤 트리거: total(누적 성공 수) 기준
- 첫 스크롤: total == 15
- 이후: 15의 배수(15, 30, 45, …)마다 발생
- 스크롤 증분: 논리 해상도 기준 720px (camY 증가 방식)
- 스크롤 대상: 월드 객체(배경/스택/낙하 빵)만 해당
- 스크린 고정 요소(캐쳐/HUD 등)는 스크롤 영향 없음
- 픽셀 높이 기준 스크롤 금지 (total 기준과 혼용 방지)


5. 스테이지 시스템

5.1 진행 조건
- 스테이지 파일에 지정된 개수(클리어 조건) 달성 시 다음 스테이지로 진행
- 클리어 시 "Stage N" 텍스트 0.5초 출력

5.2 난이도 증가 항목
- 빵 낙하 속도
- 캐쳐 이동 속도
- 스테이지 클리어 조건 빵 개수

5.3 스테이지 파일 관리
- 별도 JSON 파일로 파라미터화
- 엑셀 파일로 수정 가능
- 수정한 엑셀 파일은 배치 파일로 JSON 변환 가능


6. 설정(Options) 명세

6.1 BGM On/Off
- Off: BGM 볼륨 0 또는 재생 정지
- On: 기본 볼륨으로 복귀 (예: 0.8)

6.2 FX On/Off
- Off: 효과음 재생 금지 (또는 볼륨 0)
- On: 기본 FX 볼륨 (예: 1.0)

6.3 화면(Display)
- displayMode: Windowed / Fullscreen
- 값 변경 즉시 반영 (윈도우 모드 변경)
- 실패 시 이전 값 롤백

- 캐치 로그
