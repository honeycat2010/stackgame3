stackgame_3 구현 목표 (PR 단계별) - 기준 문서: stackgame_2_v2.1.txt

핵심 고정 규칙
- Logical 480x720 단일 기준, 레터박스/스케일은 렌더 단계에서만 처리
- 판정은 world-space, 렌더는 screen-space(screenY = worldY - camY)로 분리
- 교차 판정: prevWorldBottomY < worldLineY && currWorldBottomY >= worldLineY
- worldLineY = catcherScreenLineY + camY
- 스크롤: total 15배수에서만 camY += 720
- InGame 단일 분기: Running / Pause / ResumeCountdown
- GameOver: 진입 후 3초 입력 무시, 이후 메뉴 입력 활성화
- 캐쳐 애니메이션은 렌더 전용(판정/히트맵/월드좌표와 분리)

구현 항목

- 상태 전환 기본 경로 + Pause에서 Title/Quit 예외 경로
- 공통 타이머/시간 유틸

완료 기준
- 타이틀/메뉴/인게임/종료 전환 안정 동작

[PR 6] 스택/스크롤/카운터 생명주기

[PR 9] 캐쳐 스프라이트 애니메이션(v2.1 신규)
목표
- 상태별 2프레임 애니메이션 규칙 구현(렌더 전용)

구현 항목
- 리소스 구조: idle(0/1), move(0/1), miss(0/1)
- Running: 이동 중 move 0<->1 토글(고정 fps), 정지 중 idle frame=0 고정
- Pause/ResumeCountdown: 애니메이션 정지, idle frame=0 유지
- miss: miss 스프라이트 사용(단순 구현 시 frame=0 고정 허용)
- 프레임 인덱스 0/1 범위 강제
- 애니메이션 타이머는 InGame에서만 증가

완료 기준
- 애니메이션이 판정/좌표계/camY와 독립 동작
