빵 스택 게임 기획서 v1.3 (C++ / Steam 배포 전제)

0. 목표
- C++로 제작, Windows Steam 배포 가능 구조
- 화면비 2:3 세로형 유지 (레터박스 허용)
- 상태 흐름 고정: Title → Menu → InGame → GameOver
- 설정 메뉴 항목: BGM On/Off, FX On/Off, 화면(window, fullscreen)
- 디버그/밸런싱을 위해 스테이지 파일을 별도 파일로 관리

1. 해상도/화면 정책 (2:3 고정)

1.1 내부 기준 해상도(논리 좌표)
- 기본 출력: 480×720 (2:3)
- 모든 UI/게임 좌표는 이 내부 해상도를 기준으로 작성
- 빵이 스택되면서 일정 높이 이상 오면 빵의 가장 윗부분과 HUD만 남기고 화면을 스크롤
- 스크롤 되는 부분은 배경화면, 스택된 빵

1.2 출력 방식
- 윈도우 크기 = 480*scale × 720*scale
- 화면비가 다른 디스플레이에서 전체화면 시:
  - 레터박스(좌우 또는 상하 여백) 허용
  - 게임 영역은 항상 2:3 유지

1.3 디스플레이 모드
- displayMode: Windowed, Fullscreen(설정에서 변경 가능)

2. 상태(씬) 정의

2.1 Title
- 표시: 게임 타이틀 로고/텍스트
- 입력:
  - Enter/Space: Menu로 이동
  - Esc: 종료

2.2 Menu
- 항목: 게임 시작 / 설정 / 종료
- 이동: ↑↓, 선택: Enter, 뒤로: Esc(Title로)

2.3 Settings (Menu 내부 서브 화면)
- 항목 3개만
  1) BGM: On/Off
  2) FX: On/Off
  3) 화면: DisplayMode
- DisplayMode: Windowed / Fullscreen

- 조작:
    - ↑↓: 항목 이동
    - ←→: 값 변경 (토글/옵션 순환)
    - Enter: (필요 시) 적용
    - Esc: 저장 후 Menu로 복귀

- 저장 정책:
  - 값 변경 즉시 반영(사운드/화면)
  - Settings 종료 시 config.json 저장

2.4 InGame
- HUD:
  - Stage, PlacedCount/Target(targetPlaced)
  - total (스택된 빵의 개수)
  - miss (현재 miss한 수/3) : miss 3회 시 게임 종료

- 입력:
  - Left/Right : 캐쳐 X축 자유 이동
  - Esc: 일시정지 메뉴(선택)

- 게임 루프:
  - 빵 스폰 → 낙하 → 캐치/미스 판정 → 탑 적재 → 일정 탑 적재 시 화면 스크롤 → 스테이지 진행/게임오버

[추가 명세] InGame 일시정지 규칙
- Esc 입력 시 InGame 내부 모달 Pause 상태로 전환한다. 
- Pause 상태에서는:
  - 빵 스폰 타이머 정지
  - 낙하 업데이트 정지
  - 스택/스크롤 업데이트 정지
- 입력은 Pause 메뉴에만 전달된다.
- Resume 선택 시 3초 카운트 후, InGame 상태로 복귀한다.
- Pause는 별도 Scene으로 분리하지 않는다.
- 목적: 상태 구조 단순 유지

2.5 GameOver
- 표시: Game Over(png 파일 출력 3초 후 메뉴 이동), 최종 점수/스테이지 출력(랭킹 시스템 도입 시)
- 메뉴:
  - 재시작(InGame 새로 시작)
  - 타이틀로(Title)
  - 종료

3. 게임 규칙 (자유 이동 + 스폰 레인)

3.1 레인 시스템 (스폰 전용)
- 레인은 총 5개이며 캐쳐 이동 기준이 아니다.
- 레인은 오직 빵 스폰 X 좌표 프리셋 배열이다.
- 내부 정의:
  - 화면 폭 기준 균등 분포(레인 중심)
- 스폰 시 랜덤 레인을 선택하여 해당 X에서 낙하한다.

3.2 캐쳐 이동 (자유 이동)
- 캐쳐는 연속 좌표 기반 이동을 사용한다.
- 입력:
  - Left / Right 키만 사용

3.3 캐치 판정 (겹침 판정)
- 빵이 캐쳐의 정해진 구역에 충돌하면 겹침 판정
- 히트맵으로 충돌 판정 부분 설정 R 255, G 0, B 255

[추가 명세] 캐치 판정 정의
캐치 판정은 단일 점 판정으로 한다.
판정점은 **빵 이미지의 중심점(centerX, centerY)**이다.
centerX = imageWidth / 2
centerY = imageHeight / 2
빵 중심점을 월드 좌표로 변환한 후, 캐쳐 히트맵 좌표계로 변환하여 검사한다.
해당 좌표의 히트맵 색상이 **(R255, G0, B255)**이면 캐치 성공으로 처리한다. 
캐치 판정은 빵이 캐쳐 기준선에 도달한 프레임에 1회만 수행한다.
목적: 매 프레임 중복 판정 및 좌표계 혼용 방지

3.4 적재 위치 기록
- 캐치 성공 시 탑에 블록을 추가한다.
- 추가된 블럭수를 카운트.
- 카운트 된 수가 일정 수치가 넘어가면 화면 스크롤

[추가 명세] 화면 스크롤 기준 고정
- 스크롤 트리거는 total(누적 성공 수) 기준으로 한다.
- 첫 스크롤은 total == 15에서 발생한다.
- 이후 스크롤은 15의 배수(15, 30, 45, …)마다 증가 적용한다.
- 스크롤 계산에 “현재 높이(pixel)” 기준을 사용하지 않는다.
- 스크롤은 월드(배경 + 스택된 빵)에만 적용되며 HUD에는 적용되지 않는다. 
- 목적: total 기준과 높이 기준 혼용 방지

4. 스테이지 시스템

4.1 진행 조건
- 별도의 스테이지 파일에 지정된 개수(스테이지 클리어 조건)를 만족하면 다음 스테이지로
- 스테이지 클리어 조건을 만족하면 “Stage N” 텍스트 0.5초 출력

4.2 난이도 증가 항목
- 빵이 떨어지는 속도
- 캐쳐의 움직임
- 스테이지 클리어 조건에 필요한 빵의 개수

4.3 스테이지 별도 파일 관리
- 스테이지 난이도 조절이 쉽도록, 별도 스테이지 파일로 관리
- json 파일로 패러미터 화
- 엑셀 파일로 수정할 수 있게.
- 수정한 엑셀 파일은 배치 파일로 변환 가능하도록

5. 설정(Options) 명세

5.1 BGM On/Off
- Off: BGM 볼륨 0 또는 재생 정지
- On: 기본 볼륨으로 복귀(예: 0.8)

5.2 FX On/Off
- Off: 효과음 재생 금지(또는 볼륨 0)
- On: 기본 FX 볼륨(예: 1.0)

5.3 화면(Display)
- displayMode: Windowed / Fullscreen
  - 값 변경 즉시 반영(윈도우 모드 변경)
  - 실패 시 이전 값 롤백(안전)
