빵 스택 게임 기획서 v2.1 (C++ / Steam 배포 전제)

코드 구현할 때 주의할 것 : 코드를 너무 복잡하게 만들면 나중에 수정, 확인이 어려워짐. 코드 작업하면서 만든 함수를 별도 파일(게임
  구현과는 상관 없는 확인용 파일로, 함수와 주석을 한국어로 넣어서) reference.txt파일로 별도 저장할 것.

[ 구현 불변 조건 8개 ]
1) Logical 해상도는 480×720 단일 기준
 - 모든 좌표/레이아웃은 480×720 논리 좌표에서만 정의한다. 
 - 레터박스/스케일은 렌더 단계에서만 처리.

2) camY는 “월드 렌더 변환”에만 쓰인다
 - 월드 렌더에서만 screenY = worldY - camY.
 - camY를 물리/판정/입력 쪽에서 직접 만지지 않는다. 
- 교차 판정은 world-space bottomY로 수행한다: prevWorldBottomY < worldLineY && currWorldBottomY >= worldLineY. worldLineY는 해당 프레임의 (catcherScreenLineY + camY)로 계산한다.

3) 캐쳐/HUD/Pause/GameOver는 screen-space 고정
 - 캐쳐 Y, HUD, 모달 UI는 camY 영향 0. 
 - “스크롤하면 캐쳐가 같이 올라간다”가 보이면 즉시 버그로 간주.

4) 캐치 판정 트리거
 - prevWorldBottomY < worldLineY && currWorldBottomY >= worldLineY 조건을 만족하는 프레임으로 정의한다.
 - 빵 worldBottomY가 worldLineY를 교차한 프레임에만 판정.
 - 같은 빵에서 캐치/미스 판정은 한 번만(중복 금지 플래그).

5) 캐치 성공 조건은 “중심점 1픽셀 히트맵 (255,0,255)”
 - 판정점: centerX=imageW/2, centerY=imageH/2 
 - 월드→히트맵 좌표 변환 후 색상 체크.
 - AABB, 면적, 다중 샘플링 같은 확장 금지(필요하면 v2).

6) 스크롤 트리거는 total 15배수만
 - 스크롤 트리거는 total 기준: 15, 30, 45… 
 - “픽셀 높이 기반” 스크롤은 금지. 
 - 스크롤 증분은 camY += 720 고정. 

7) Pause는 InGame 내부 모달, 업데이트는 완전 정지
 - Pause 동안: 스폰/낙하/스택/스크롤 업데이트 전부 정지. 
 - Ingame에서 ESC 키를 누르면 Pause 메뉴 출력
 - Pause 메뉴는 Resume / Title / Quit
   - 각 메뉴 기능
   - Resume : 게임 재게 (Resume 버튼을 누르면 3초 카운트 후 게임 재게)
   - Title : 타이틀 화면으로
   - Quit : 게임 종료
   - Pause 메뉴에서 Title / Quit 버튼을 누르면 팝업창으로 "진행중인 상황이 리셋됩니다. 진행하시겠습니까?" "예/아니오" 출력. 예: Title / Quit 기능 실행. 아니오 : Pause 메뉴로 이동. 
 - 입력은 Pause 메뉴로만 라우팅.
- Pause의 확인 팝업(Title/Quit 확인: 예/아니오)은 Pause 상태 내부 UI 서브상태로 취급한다. 즉, 게임 상태머신의 상위 상태는 여전히 Pause이며 Running/ResumeCountdown으로 전환되지 않는다.
 - 별도 Scene 분리 금지. 
 - ResumeCountdown 동안 월드 업데이트는 정지 상태를 유지한다.
 - 입력은 Esc만 허용하며, Esc 입력 시 Pause 상태로 복귀한다.

8) 업데이트 진입점은 항상 “단일 스위치”
 - InGame update()는 시작에서 상태를 딱 한 번만 분기한다:
 - Running / Pause / ResumeCountdown
 - 상태별 로직이 프레임 중간중간 섞이는 순간부터 꼬이기 시작한다.


0. 목표
- C++로 제작, Windows Steam 배포 가능 구조
- 화면비 2:3 세로형 유지 (레터박스 허용)
- 상태 흐름 고정: Title → Menu → InGame → GameOver
- 위 고정 흐름(Title → Menu → InGame → GameOver)은 정상 플레이 루프 기준이다. 단, InGame의 Pause 메뉴에서 사용자가 Title 또는 Quit를 선택한 경우에는 예외적으로 해당 상태로 즉시 전환할 수 있다.
- 설정 메뉴 항목: BGM On/Off, FX On/Off, 화면(Windowed, Fullscreen)
- 디버그/밸런싱을 위해 스테이지 파일을 별도 파일로 관리


1. 해상도/화면 정책

1.1 내부 기준 해상도 (논리 좌표)
- 기본 출력: 480×720 (2:3)
- 모든 UI/게임 좌표는 이 내부 해상도를 기준으로 작성
- 빵이 스택되어 일정 높이 이상 오면 빵의 가장 윗부분과 HUD만 남기고 화면을 스크롤
- 스크롤 대상: 배경, 스택된 빵

1.2 출력 방식
- 윈도우 크기 = 480*scale × 720*scale
- 화면비가 다른 디스플레이에서 전체화면 시:
  - 레터박스(좌우 또는 상하 여백) 허용
  - 게임 영역은 항상 2:3 유지

1.3 디스플레이 모드
- displayMode: Windowed / Fullscreen (설정에서 변경 가능)


2. 상태(씬) 정의

2.1 Title
- 표시: 게임 타이틀 로고/텍스트
- 입력:
  - Enter/Space: Menu로 이동
  - Esc: 종료

2.2 Menu
- 항목: 게임 시작 / 설정 / 종료
- 이동: ↑↓  선택: Enter  뒤로: Esc(Title로)

2.3 Settings (Menu 내부 서브 화면)
- 항목:
  1) BGM: On/Off
  2) FX: On/Off
  3) 화면: Windowed / Fullscreen
- 조작:
  - ↑↓: 항목 이동
  - ←→: 값 변경 (토글/옵션 순환)
  - Esc: 저장 후 Menu로 복귀
- 저장 정책:
  - 값 변경 즉시 반영 (사운드/화면)
  - Settings 종료 시 config.json 저장

2.4 InGame
- HUD:
  - Stage
  - PlacedCount/Target (targetPlaced)
  - total (누적 스택 개수)
  - miss (현재 miss 수/3) : miss 3회 시 게임 종료
- 입력:
  - Left/Right: 캐쳐 X축 자유 이동
  - Esc: 일시정지 메뉴 진입
- 게임 루프:
  - 빵 스폰 → 낙하 → 캐치/미스 판정 → 탑 적재 → 일정 탑 적재 시 화면 스크롤 → 스테이지 진행/게임오버

- 일시정지(Pause) 규칙:
 - Pause 동안: 스폰/낙하/스택/스크롤 업데이트 전부 정지. 
 - Ingame에서 ESC 키를 누르면 Pause 메뉴 출력
 - Pause 메뉴는 Resume / Title / Quit
   - 각 메뉴 기능
   - Resume : 게임 재게 (Resume 버튼을 누르면 3초 카운트 후 게임 재게)
   - Title : 타이틀 화면으로
   - Quit : 게임 종료
   - Pause 메뉴에서 Title / Quit 버튼을 누르면 팝업창으로 "진행중인 상황이 리셋됩니다. 진행하시겠습니까?" "예/아니오" 출력. 예: Title / Quit 기능 실행. 아니오 : Pause 메뉴로 이동. 
 - 입력은 Pause 메뉴로만 라우팅.
- Pause의 확인 팝업(Title/Quit 확인: 예/아니오)은 Pause 상태 내부 UI 서브상태로 취급한다. 즉, 게임 상태머신의 상위 상태는 여전히 Pause이며 Running/ResumeCountdown으로 전환되지 않는다.
 - 별도 Scene 분리 금지. 

2.5 GameOver
- 표시: Game Over 이미지(png) 3초 출력 후 메뉴 표시
- GameOver 진입 후 3초 동안은 입력을 무시하며, 3초 경과 후 메뉴 입력을 활성화한다.
- 메뉴:
  - 재시작 (InGame 새로 시작)
  - 타이틀로 (Title)
  - 종료


3. 게임 규칙

3.1 레인 시스템 (스폰 전용)
- 레인은 총 5개이며 캐쳐 이동 기준이 아님
- 레인은 오직 빵 스폰 X 좌표 프리셋 배열
- 정의: 화면 폭 기준 균등 분포 (레인 중심)
- 스폰 시 랜덤 레인을 선택하여 해당 X에서 낙하

3.2 캐쳐 이동 (자유 이동)
- 캐쳐는 연속 좌표 기반 이동
- 입력: Left / Right 키만 사용

3.2.1 캐쳐 스프라이트 애니메이션
- 캐쳐 스프라이트는 상태별 2프레임(0 / 1) 구조로 구성한다.
  - idle: 0 / 1
  - move: 0 / 1
  - miss: 0 / 1
- 애니메이션은 **렌더 전용 요소**이며, 월드 좌표·판정·히트맵 계산과 무관하다.

3.2.2 상태별 사용 규칙
3.2.2.1 Running 상태
 - 캐쳐가 좌우로 이동 중일 경우:
 - move 스프라이트 사용
 - 0 ↔ 1을 고정 fps(예: 8~10fps)로 토글

- 캐쳐가 정지 상태일 경우:
 - idle 스프라이트 사용
 - frame = 0 고정 (idle 애니메이션 토글하지 않음)

3.2.2.2. Pause / ResumeCountdown 상태
- 애니메이션 정지
 - idle 스프라이트 frame = 0 유지

3.2.2.3. - miss 상태
- miss 스프라이트 사용
- 0 ↔ 1 토글
- 단순 구현을 위해 frame = 0 고정 권장

3.2.3. 기술적 제약 (꼬임 방지용)
- 애니메이션 프레임 인덱스는 0 또는 1만 사용한다.
- 애니메이션 타이머는 InGame 상태에서만 증가한다.
- camY 변경과 무관하게 캐쳐는 screen-space 고정 객체이므로, 애니메이션은 위치 계산과 독립적으로 동작한다.
- 애니메이션 상태 전환은 InGame의 단일 상태 분기(Running / Pause / ResumeCountdown)에 종속된다.

3.3 캐치 판정
- 판정 방식: 단일 점 판정
- 판정점: 빵 이미지의 중심점 (centerX = imageWidth/2, centerY = imageHeight/2)
- 판정 절차:
  1. 빵의 worldBottomY가 worldLineY를 교차한 프레임(prevWorldBottomY < worldLineY && currWorldBottomY >= worldLineY)에 판정 1회 수행
  2. 빵 중심점을 월드 좌표로 변환 후 캐쳐 히트맵 좌표계로 변환
  3. 해당 좌표의 히트맵 색상이 (R255, G0, B255)이면 캐치 성공
- 매 프레임 중복 판정 금지 (좌표계 혼용 방지)
- 판정과 렌더의 책임 분리: 판정은 world-space 기준으로만 수행하고, 렌더는 별개로 screenY = worldY - camY를 사용한다.
- 프레임 순서 고정: 교차 판정 계산(prev/curr/worldLineY)을 먼저 수행하고, 같은 프레임의 camY 변경(스크롤 적용)은 그 다음에 수행한다.

3.4 적재 위치 기록
- 캐치 성공 시 탑에 블록 추가, 누적 카운트(total) 증가
- total이 15의 배수가 될 때마다 화면 스크롤 발생 (→ 4.3 스크롤 동작 참조)

3.4.1. total 정의(역할 + 생명주기)
- total은 게임 런 전체에서 성공적으로 스택된 빵의 누적 개수이며, 화면 스크롤 트리거(15의 배수) 계산에만 사용한다. 
- total은 스테이지 전환 시 유지되며, 새 게임 시작(재시작) 시에만 0으로 초기화된다.
3.4.2. Placed/Target 정의(스테이지 클리어 전용)
- PlacedCount/Target은 현재 스테이지 클리어 조건이며, 스테이지 시작 시 0으로 리셋된다.

4. 좌표계 및 스크롤 정책

4.1 좌표계 구분
- World 좌표 적용 객체:
  - 스택된 빵
  - 낙하 중 빵
  - 배경
  - 렌더 시에만 screenY = worldY - camY 적용

- Screen 좌표 고정 객체:
  - 캐쳐
  - HUD
  - Pause UI
  - GameOver UI
  - camY의 영향을 받지 않음

4.2 camY 적용 규칙
- camY는 월드 객체에만 적용
- 캐쳐 및 모든 UI 요소에는 camY 적용 안 함
- ScreenToWorld 보정 계산을 캐쳐에 사용하지 않음
- camY 값 변경은 스크롤 트리거(total 15의 배수)에서만 발생

4.3 스크롤 동작
- 스크롤 트리거: total(누적 성공 수) 기준
- 첫 스크롤: total == 15
- 이후: 15의 배수(15, 30, 45, …)마다 발생
- 스크롤 증분: 논리 해상도 기준 720px (camY 증가 방식)
- 스크롤 대상: 월드 객체(배경/스택/낙하 빵)만 해당
- 스크린 고정 요소(캐쳐/HUD 등)는 스크롤 영향 없음
- 픽셀 높이 기준 스크롤 금지 (total 기준과 혼용 방지)


5. 스테이지 시스템

5.1 진행 조건
- 스테이지 파일에 지정된 개수(클리어 조건) 달성 시 다음 스테이지로 진행
- 클리어 시 "Stage N" 텍스트 0.5초 출력
- 게임 화면 유지 : 스테이지 단계만 올라감 (관련해 스테이지 컨피그 파일에 미리 지정된 스테이지 난이도도 함께 올라감)

5.2 난이도 증가 항목
- 빵 낙하 속도
- 캐쳐 이동 속도
- 스테이지 클리어 조건 빵 개수

5.3 스테이지 파일 관리
- 별도 JSON 파일로 파라미터화
- 엑셀 파일로 수정 가능
- 수정한 엑셀 파일은 배치 파일로 JSON 변환 가능


6. 설정(Options) 명세

6.1 BGM On/Off
- Off: BGM 볼륨 0 또는 재생 정지
- On: 기본 볼륨으로 복귀 (예: 0.8)

6.2 FX On/Off
- Off: 효과음 재생 금지 (또는 볼륨 0)
- On: 기본 FX 볼륨 (예: 1.0)

6.3 화면(Display)
- displayMode: Windowed / Fullscreen
- 값 변경 즉시 반영 (윈도우 모드 변경)
- 실패 시 이전 값 롤백


7. 디버그 로그
- 스폰 로그
- 캐치 로그
- 오버행 로그
- COM 로그

7.1 판정/스크롤 디버그 필수 필드
- 캐치 판정 로그: breadId, prevWorldBottomY, currWorldBottomY, worldLineY, centerWorldX, centerWorldY, hitmapX, hitmapY, hitColor(R,G,B), judgedResult(CATCH/MISS), judgedFlagBefore/After
- 스크롤 로그: totalBefore, totalAfter, scrollTriggered(true/false), camYBefore, camYAfter
- Pause/Resume 로그: pauseEnterTime, resumeCountdownStart, resumeCountdownEnd, stateTransition(Running/Pause/ResumeCountdown)
- 로그 레벨 권장: INFO(상태 전환), DEBUG(좌표/판정값), WARN(규칙 위반/예외)
