# reference.txt
# PR1~PR5에서 만든 함수 확인용 문서 (게임 구현과 분리)

[RenderLayout.cpp]
1) ViewportRect CalculateLetterboxViewport(int windowWidth, int windowHeight, int logicalWidth, int logicalHeight)
- 역할: 윈도우 크기와 논리 해상도 비율을 비교해 레터박스 viewport를 계산한다.

[InputRouter.cpp]
2) InputAction ParseRawInput(const std::string& raw)
- 역할: 문자열 입력을 공통 InputAction으로 변환한다.
3) bool IsAllowedInTitle(InputAction action)
4) bool IsAllowedInMenu(InputAction action)
5) bool IsAllowedInInGame(InGameSubState subState, InputAction action)
6) bool IsAllowedInGameOver(InputAction action)
7) InputAction InputRouter::Route(AppScene scene, InGameSubState subState, const std::string& raw) const
- 역할: 씬/서브상태 규칙을 적용해 허용 입력만 통과시킨다.

[GameApp.cpp]
8) bool GameApp::HandleInGameTickCommand(const std::string& line)
- 역할: tick <sec> 명령을 파싱해 InGame 시뮬레이션을 진행한다.

9) void GameApp::UpdateInGameSimulation(double dtSec)
- 역할: PR5 핵심 루프(스폰/낙하/교차 판정/캐치/미스)를 수행한다.
- 규칙:
  - 교차 조건: prevBottomY < worldLineY && currBottomY >= worldLineY
  - 같은 빵은 judged 플래그로 1회만 판정
  - center point -> hitmap 좌표 변환 후 (255,0,255)면 캐치
  - 미스 3회 이상이면 GameOver 요청

10) bool GameApp::IsCatchHitByCenterPoint(const ActiveBread& bread, int& outHitmapX, int& outHitmapY, Rgb& outColor) const
- 역할: 빵 중심점을 캐쳐 히트맵 좌표로 변환하고 캐치 성공 여부를 반환한다.
- 주의:
  - 판정은 world-space 업데이트 흐름에서 호출하지만, 히트맵 샘플은 screen-space 좌표로 계산한다.

11) GameApp::Rgb GameApp::SampleCatcherHitmap(int hitmapX, int hitmapY) const
- 역할: 히트맵 색 샘플링 함수(현재는 테스트용 단순 마젠타 영역 구현).
- 주의:
  - PR5 단계에서는 실제 이미지 로딩 대신 고정 규칙으로 동작한다.

12) void GameApp::SpawnBread()
- 역할: 5레인 중 하나에서 빵을 생성한다.

13) void GameApp::ResetInGameSession()
- 역할: InGame 시작/재시작 시 상태를 초기화한다.

14) std::vector<double> GameApp::BuildSpawnLanes() const
- 역할: 논리폭 480 기준 5개 스폰 레인 중심 좌표를 생성한다.

15) void GameApp::PrintInGameHudFrame() const
- 역할: Stage/Placed/Target/total/miss 및 간단 상태를 표시한다.

[SceneManager.cpp]
16) void SceneManager::ApplyIfRequested()
- 역할: 요청된 씬 전환을 현재 씬에 반영한다.
